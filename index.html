What is agent.md?

A mini‑manifest that lives in your repo root. When an LLM‑powered agent spins up, it loads this file into the system part of its prompt, giving it permanent knowledge of your project's rules, tools and style.

Four Key Benefits

1

Single source of truth

Directory layout, naming, CI steps stay consistent across agents & humans.

2

Higher LLM accuracy

Codex scored +5 pp when an AGENTS.md was present (75% → 80%).

3

Zero‑friction onboarding

New contributors "read the manual" in one file; agents do the same in milliseconds.

4

Built‑in guard‑rails

Embed commands like npm run lint && npm audit; the agent must pass before finishing.
